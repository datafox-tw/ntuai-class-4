在現在的資料夾裡面，有許多個agent範例，這些範例包含memory(長期記憶）, knowledge(資料庫知識), tool use(web search, 文件管理,youtube agent等) 我希望透過一個冰島旅行文件管理的一個網頁讓學生學習剛剛使用的那些example(1~9)如果放在一個真實的專案裡面會長怎樣，我希望不只是強調ai agent功能，而是希望學生能知道，除了一個問答的聊天界面，還有很多軟體功能可以結合agent的專長，讓一個網站可以又多功能又穩定還很清楚。希望學生可以開箱及用，因為他們可能呆呆地不會設置一些東西

我整理完需求與現有範例了，接下來會直接做成一個可跑的教學專案，不只聊天而是完整網站流程。
實作計畫如下：

建立一個「零額外安裝」的單檔 web app（Python 內建 HTTP server）
首頁做成「Example 1~9 對照真實產品功能」導覽，讓學生看得懂每個範例在產品裡扮演什麼角色
做文件管理與 knowledge 功能：掃描 iceland_data_pdf、建立本地索引、可提問
做 memory 功能：對話紀錄 + 旅客偏好摘要（SQLite 長期保存）
做 tool use 功能：web search、YouTube 字幕摘要、檔案搜尋、照片素材瀏覽
提供一鍵啟動（python iceland_travel_lab.py），沒有 API key 也能跑；若未來補 key 再接進 LLM
補一份簡短教學文件，讓學生開箱就能使用與理解架構
